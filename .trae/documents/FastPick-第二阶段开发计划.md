# FastPick 第二阶段开发计划

## 项目概述
在已完成的 UI 框架基础上，实现图片加载、评级、预删除等核心功能，将"空壳 UI"变成"可用应用"。

---

## 一、核心决策回顾

| 决策项 | 最终方案 | 理由 |
|--------|----------|------|
| 图片加载 | 一次性加载路径 + 虚拟滚动加载缩略图 | 路径扫描仅一次，虚拟滚动保证流畅 |
| 缩略图生成 | 系统缓存优先 → WIC 实时生成 | 速度快，内存占用低 |
| RAW 解码 | WIC 为主 + ExifTool 为辅 | 兼容性优先，内嵌预览优先 |
| 元数据写入 | 统一使用 ExifTool | 支持所有格式，行业标准 |
| 数据持久化 | 无数据库，元数据写入文件本身 | 轻量可靠，避免数据不一致 |
| 预删除显示 | 保留显示 + 红色角标 | 直观可见，支持筛选 |
| 多选评级 | 批量设置相同评级 | 符合摄影筛选习惯 |

## 二、开发阶段划分

### 阶段 2.1：图片加载与解码（预计 1 周）

#### 任务 2.1.1：图片扫描服务
- **目标**：实现双路径图片扫描和路径管理
- **内容**：
  - 创建 `Services/ImageScanService.cs`
  - 扫描两个文件夹，获取所有 JPG/RAW 文件路径
  - 按文件名匹配 JPG/RAW，生成虚拟图片项列表
  - 路径失效检测和提示
- **交付物**：
  - `ImageScanService` 类
  - `PhotoItem` 模型类（文件名、JPG路径、RAW路径、评级、预删除标记）

#### 任务 2.1.2：缩略图生成服务
- **目标**：实现缩略图生成和缓存
- **内容**：
  - 创建 `Services/ThumbnailService.cs`
  - 优先读取 Windows 系统缩略图缓存（IShellItemImageFactory）
  - 系统缓存不存在时，用 WIC 异步生成（100×80）
  - 实现 LRU 内存缓存（最多 200 张）
- **交付物**：
  - `ThumbnailService` 类
  - 缩略图缓存管理器

#### 任务 2.1.3：虚拟滚动实现
- **目标**：实现缩略图区虚拟滚动
- **内容**：
  - 使用 ItemsRepeater + UniformGridLayout
  - 仅渲染可视区域的缩略图
  - 滚动时动态加载新区域，卸载离开区域的图片
- **交付物**：
  - 虚拟滚动缩略图列表
  - 流畅的滚动体验（2000+ 张图片）

#### 任务 2.1.4：预览图加载
- **目标**：实现单图/多图预览
- **内容**：
  - JPG 优先加载原图
  - RAW 优先提取内嵌 JPG 预览
  - 切换图片时释放前一张预览图内存
  - 多选时平铺显示前 5 张
- **交付物**：
  - 预览图加载器
  - 内存释放机制

## 三、技术实现细节

### 3.1 核心类设计

```
Models/
├── PhotoItem.cs              # 虚拟图片项
├── FileTypeEnum.cs           # 文件类型枚举
└── DeleteOptionEnum.cs       # 删除选项枚举

Services/
├── ImageScanService.cs       # 图片扫描服务
├── ThumbnailService.cs       # 缩略图生成服务
├── MetadataService.cs        # 元数据读写服务
├── FileOperationService.cs   # 文件操作服务
└── SettingsService.cs        # 设置服务

ViewModels/
└── MainViewModel.cs          # 主视图模型
```

### 3.2 关键技术点

| 技术点 | 实现方案 |
|--------|----------|
| 虚拟滚动 | ItemsRepeater + UniformGridLayout |
| 缩略图缓存 | LRU 算法，最多 200 张 |
| RAW 预览 | 优先提取内嵌 JPG 预览 |
| 元数据写入 | ExifTool 命令行调用 |
| 删除到回收站 | Windows API (SHFileOperation) |
| 配置持久化 | appsettings.json + Windows 注册表 |

### 3.3 内存控制策略

- 缩略图缓存：最多 200 张（约 6.4MB）
- 预览图卸载：切换时释放内存
- RAW 降级：优先使用内嵌预览（2-5MB）
- 内存监控：超过 200MB 时清空缩略图缓存

## 四、错误处理策略

| 错误场景 | 处理方案 |
|----------|----------|
| 图片损坏 | 显示占位图 + 提示，保留条目 |
| 路径失效 | 标题栏红色提示 + 手动刷新 |
| 元数据写入失败 | Toast 提示 + 日志记录 + 重试按钮 |
| 文件被占用 | 重试 2 次，仍失败则提示用户 |

## 七、风险与应对

| 风险 | 应对措施 |
|------|----------|
| ExifTool 集成复杂 | 提前测试命令行调用，准备 fallback 方案 |
| RAW 解码性能差 | 严格使用内嵌预览，完整解码异步延迟 |
| 内存占用过高 | LRU 缓存 + 内存监控 + 主动释放 |
| 虚拟滚动卡顿 | 优化 ItemsRepeater 配置，减少重绘 |
