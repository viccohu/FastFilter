# FastPick 黑色主题实现计划

## 项目概述
为 FastPick 应用实现 Fluent Design 黑色主题，提供深色沉浸式的看图选片体验。

---

## 一、当前状态分析

- ✅ WinUI 3 项目已创建
- ✅ 五段式 UI 界面已完成
- ✅ 无边框窗口 + 标题栏拖拽已实现
- ⏳ 需要实现黑色主题

---

## 二、主题设计规范

### 2.1 色彩系统

| 用途 | 色值 | 说明 |
|------|------|------|
| 主背景 | `#0D0D0D` | 页面主背景（略纯黑，减少视觉疲劳） |
| 次级背景 | `#1E1E1E` | 卡片、面板、抽屉背景 |
| 三级背景 | `#2D2D2D` | 按钮、输入框背景 |
| 悬停背景 | `#383838` | 控件悬停状态 |
| 按下背景 | `#444444` | 控件按下状态 |
| 主文本 | `#FFFFFF` | 主要内容文本 |
| 次级文本 | `#B0B0B0` | 辅助说明文本 |
| 禁用文本 | `#666666` | 禁用状态文本 |
| 强调色 | `#0078D4` | Fluent 标准蓝（主按钮、选中状态） |
| 强调色悬停 | `#1089FF` | 悬停时高亮 |
| 危险色 | `#D13438` | 删除按钮 |
| 成功色 | `#107C10` | 文件信息按钮 |
| 边框 | `#333333` | 分割线、边框 |
| 选中边框 | `#0078D4` | 选中状态高亮边框 |

### 2.2 区域配色方案

```
┌─────────────────────────────────────────────────────────────┐
│ 标题栏: #1E1E1E (次级背景)                                   │
├─────────────────────────────────────────────────────────────┤
│ 抽屉区: #1E1E1E (次级背景) + 底部边框 #333333               │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│ 预览区: #0D0D0D (主背景)                                    │
│                                                             │
├─────────────────────────────────────────────────────────────┤
│ 缩略图区: #1E1E1E (次级背景)                                │
├─────────────────────────────────────────────────────────────┤
│ 底栏: #1E1E1E (次级背景)                                    │
└─────────────────────────────────────────────────────────────┘
```

## 三、实现步骤

### 阶段一：创建主题资源字典（预计 1 天）

#### 任务 1.1：创建主题资源文件
- **目标**：创建 `Themes/DarkTheme.xaml` 资源字典
- **内容**：
  - 定义所有颜色资源（SolidColorBrush）
  - 定义按钮样式（TitleBarButtonStyle, PrimaryButtonStyle, DangerButtonStyle）
  - 定义输入框样式（TextBoxStyle）
  - 定义抽屉样式（DrawerStyle）
  - 定义边框样式（BorderStyle）

#### 任务 1.2：创建主题管理器
- **目标**：创建 `Services/ThemeService.cs`
- **内容**：
  - 主题切换逻辑
  - 监听系统主题变化
  - 保存用户主题偏好

### 阶段二：修改 App.xaml 全局主题（预计 0.5 天）

#### 任务 2.1：引入主题资源
- **目标**：修改 `App.xaml`
- **内容**：
  - 引入 DarkTheme.xaml 资源字典
  - 设置默认主题为 Dark
  - 配置全局样式覆盖

### 阶段三：修改 MainPage.xaml 区域样式（预计 2 天）

#### 任务 3.2：抽屉区样式

#### 任务 3.3：预览区样式

#### 任务 3.4：缩略图区样式

- **内容**：

  - 缩略图项背景 `#2D2D2D`
  - 选中状态边框 `#0078D4`
  - 预删除角标颜色调整
  - 星级控件颜色调整

#### 任务 3.5：底栏样式
- **目标**：修改底栏背景和功能按钮
- **内容**：
  - 背景色 `#1E1E1E`
  - 文件信息按钮样式
  - 缩放控件样式

- **目标**：修改文件信息抽屉
- **内容**：
  - 背景色使用 Acrylic 毛玻璃效果

  - 标题栏绿色背景调整

### 阶段四：控件状态样式（预计 1 天）

#### 任务 4.1：按钮状态样式
- **目标**：实现按钮的 4 种状态样式
- **内容**：
  - Normal: 背景 `#2D2D2D`
  - PointerOver: 背景 `#383838`
  - Pressed: 背景 `#444444`

- **内容**：

#### 任务 4.3：单选框和复选框样式
- **目标**：调整选择控件颜色
- **内容**：

## 四、技术实现要点

### 4.1 资源字典结构
```
Themes/
├── DarkTheme.xaml          # 深色主题资源
├── LightTheme.xaml         # 浅色主题资源（预留）
└── ThemeResources.xaml     # 共享资源
```

### 4.2 关键样式定义
- **PageBackground**: `#0D0D0D`
- **SurfaceBackground**: `#1E1E1E`
- **ElevatedBackground**: `#2D2D2D`
- **AccentColor**: `#0078D4`
- **TextPrimary**: `#FFFFFF`
- **TextSecondary**: `#B0B0B0`

### 4.3 Acrylic 效果配置
```xml
<AcrylicBrush x:Key="AcrylicBackground"
    BackgroundSource="HostBackdrop"
    TintColor="#1E1E1E"
    TintOpacity="0.8"
    FallbackColor="#1E1E1E"/>
```

## 六、风险与注意事项

### 6.1 潜在风险
1. **Acrylic 性能问题**：在低端设备上可能造成卡顿
2. **颜色对比度**：需要确保文本可读性符合 WCAG 标准
3. **系统兼容性**：不同 Windows 版本 Acrylic 效果可能有差异

### 6.2 开发建议
1. 先实现基础颜色，再添加 Acrylic 效果
2. 使用热加载实时预览样式变化
3. 在不同 DPI 设置下测试显示效果
4. 预留浅色主题扩展接口
