# FastPick UI 开发计划

## 项目概述
FastPick 是一款 Windows 看图选片应用，采用 **WinUI 3 + Fluent UI** 设计规范，实现五段式界面布局（标题栏 - 抽屉区 - 预览区 - 缩略图区 - 底部工具栏）。

本计划专注于**界面开发阶段**，业务逻辑将在界面完成后绑定。

---

## 一、当前项目状态

- ✅ WinUI 3 项目已创建（.NET 9）
- ✅ 项目可编译运行
- ⏳ 需要实现五段式 UI 界面

---

## 二、UI 开发阶段规划

### 阶段一：基础布局框架（预计 3 天）

#### 任务 1.1：创建主窗口布局
- **目标**：搭建五段式垂直布局框架
- **内容**：
  - 修改 MainWindow.xaml，创建 5 行 Grid 布局
  - Row 0: 标题栏（固定高度 48px）
  - Row 1: 抽屉区（动态高度，可展开/收起）
  - Row 2: 预览区（自适应填充）
  - Row 3: 缩略图区（固定高度 200px）
  - Row 4: 底部工具栏（固定高度 40px）

#### 任务 1.2：实现自定义标题栏
- **目标**：创建 Fluent UI 风格的自定义标题栏
- **内容**：
  - 左侧：路径、导出、删除、筛选、设置 按钮
  - 右侧：最小化、最大化、关闭按钮
  - 支持窗口拖拽
  - 按钮样式符合 Fluent UI 规范

#### 任务 1.3：实现抽屉区容器
- **目标**：创建可展开/收起的抽屉容器
- **内容**：
  - 使用 Expander 或自定义 Panel 实现抽屉效果
  - 默认收起状态（高度 0）
  - 点击标题栏按钮时展开对应抽屉
  - 抽屉切换动画（300ms 滑动）

---

### 阶段二：各抽屉界面实现（预计 4 天）

#### 任务 2.1：路径抽屉
- **目标**：实现双路径选择界面
- **内容**：
  - 路径1 输入框 + 选择按钮
  - 路径2 输入框 + 选择按钮
  - 载入按钮（主色调 #0078D4）

#### 任务 2.2：导出抽屉
- **目标**：实现导出配置界面
- **内容**：
  - RAW 文件导出控制下拉框（5星以上/4星以上/...）
  - 导出文件夹名配置（JPG/RAW 输入框）
  - 导出路径输入框 + 浏览按钮
  - 导出按钮

#### 任务 2.3：删除抽屉
- **目标**：实现删除确认界面
- **内容**：
  - 显示"当前预删除：[x张]"（红色文字）
  - 删除选项：同时删除 / 仅 JPG / 仅 RAW
  - 执行删除按钮

#### 任务 2.4：筛选抽屉
- **目标**：实现筛选条件界面
- **内容**：
  - 文件类型筛选：全部 / 双文件 / 仅 JPG / 仅 RAW
  - 星级筛选：有星 / 无星 / 等于 / 大于等于 / 小于等于 + 星级选择

#### 任务 2.5：设置抽屉
- **目标**：实现设置界面
- **内容**：
  - 设置项占位区域
  - 保存 / 取消按钮

---

### 阶段三：预览区实现（预计 2 天）

#### 任务 3.1：单图预览模式
- **目标**：实现单张图片预览
- **内容**：
  - Image 控件居中显示
  - 支持 Uniform 缩放模式
  - 加载进度环占位

#### 任务 3.2：多图平铺模式
- **目标**：实现多选平铺预览（最多5张）

  - ItemsRepeater 或 Grid 布局
  - 5列1行网格
  - 每张图最小尺寸 200×150px
  - 灰色边框 1px

---

### 阶段四：缩略图区实现（预计 3 天）

#### 任务 4.1：缩略图网格布局
- **目标**：实现横向滚动缩略图网格
- **内容**：
  - GridView 或 ItemsRepeater 实现网格
  - 8列2行布局
  - 横向滚动支持
  - 间距 5px

#### 任务 4.2：缩略图项模板
- **目标**：创建缩略图项数据模板
- **内容**：
  - 缩略图 Image（100×80px）
  - 预删除红色角标（右上角，15×15px）
  - 星级 RatingControl（左下角）
  - 文件类型标签（右下角，JPG/RAW/双格式）
  - 选中状态蓝色边框（2px）

#### 任务 4.3：缩略图交互
- **目标**：实现缩略图选择交互
- **内容**：
  - 点击选中
  - Ctrl+点击多选
  - Shift+点击连续选择
  - 悬停显示文件名提示

### 阶段六：弹窗与视觉反馈（预计 2 天）

## 三、技术实现要点

### 3.1 布局技术选型
| 区域 | 推荐控件 | 说明 |
|------|----------|------|
| 整体布局 | Grid | 5行垂直布局 |
| 抽屉区 | Expander / ContentControl | 动态显示/隐藏 |
| 预览区单图 | Image | 居中显示 |
| 预览区多图 | ItemsRepeater + UniformGridLayout | 平铺布局 |
| 缩略图区 | GridView / ItemsRepeater | 网格布局 |
| 底部工具栏 | Grid | 左中右分区 |

### 3.2 样式资源规划
```
Resources/
├── Colors.xaml      # 颜色定义
├── Styles.xaml      # 控件样式
├── Templates.xaml   # 数据模板
└── Converters.xaml  # 值转换器
```

### 3.3 关键尺寸规范
- 最小窗口尺寸：1280×720px
- 标题栏高度：48px
- 缩略图区高度：200px
- 底部工具栏高度：40px
- 缩略图尺寸：100×80px
- 抽屉展开动画：300ms

## 五、风险与注意事项

### 5.1 潜在风险
1. **WinUI 3 版本兼容性**：确保使用稳定的 Windows App SDK 版本
2. **自定义标题栏**：需要正确处理窗口拖拽和按钮点击区域
3. **缩略图虚拟化**：大量图片时需要考虑 UI 虚拟化

### 5.2 开发建议
1. 先搭建空壳布局，再填充细节
2. 使用 MVVM 模式，为后续业务逻辑预留接口
3. 样式统一使用 ResourceDictionary 管理
4. 及时在真机上测试高 DPI 效果

## 六、问题确认

在开始开发前，请确认以下问题：

1. **窗口控制按钮位置**：标题栏右侧的窗口控制按钮（最小化/最大化/关闭）是否需要完全自定义，还是保留系统默认样式？

2. **抽屉动画效果**：抽屉展开/收起是否需要有滑动动画，还是直接显示/隐藏？

3. **缩略图布局**：缩略图区是固定 8列2行，还是需要根据窗口宽度动态调整列数？

4. **主题支持**：是否需要优先实现深色主题，还是先完成浅色主题？

5. **开发顺序**：是否按照上述阶段顺序开发，还是有优先级调整？
