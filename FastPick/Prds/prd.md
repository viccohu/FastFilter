Windows 看图选图应用开发文档（Trae 项目）
一、项目概述
1.1 项目背景
本项目是为摄影工作流设计的 Windows 桌面应用，核心目标是高效筛选、评级和管理相机拍摄的 JPG+RAW 双格式照片，解决传统看图软件在双格式处理、批量评级和筛选上的痛点。
1.2 核心价值
双路径加载：同时加载两个文件夹，自动识别并折叠同一张照片的 JPG/RAW
高效评级：支持批量评级，JPG 写入 Exif，RAW 写入 .xmp 元数据
智能删除：预删除标记 + 批量删除，可选择仅删 JPG/RAW 或全部
快速导出：按评级筛选导出，自动分文件夹存放 JPG/RAW
流畅体验：基于 WIC 解码，支持 2000+ 张图片同时加载不卡顿
1.3 技术栈选型
UI 框架：WinUI 3 + Fluent UI，符合 Windows 设计语言，提供现代化界面
图片解码：Windows Imaging Component (WIC)，原生高效解码，支持常见 RAW 格式
元数据处理：ExifTool（成熟方案）或 WIC 原生 API，处理 JPG Exif 评级和 RAW .xmp 写入
文件系统：.NET System.IO + Windows API，处理双路径加载、批量操作和回收站删除
开发工具：Trae AI 代码助手 + Visual Studio，提升开发效率

四、技术方案设计
4.1 架构设计
采用 MVVM (Model-View-ViewModel) 架构，分离 UI 与业务逻辑，提升可维护性和测试性：
Model：文件系统操作、图片解码、元数据读写、批量处理核心逻辑
ViewModel：状态管理（选中图片、评级、预删除列表）、命令绑定（评级、删除、导出）
View：Fluent UI 组件，响应 ViewModel 状态变化，处理用户交互
4.2 图片加载与解码
核心依赖：Windows Imaging Component (WIC)
优势：原生高效，支持 JPG、PNG 及常见 RAW 格式（佳能 CR2、尼康 NEF、索尼 ARW 等）
实现：通过 IWICImagingFactory 创建解码器，异步加载缩略图和预览图，避免 UI 卡顿
性能优化：
懒加载：仅加载可见区域的缩略图，滚动时动态加载
缓存机制：缓存已解码的图片数据，避免重复解码
多线程：使用 Task 异步处理加载和解码，不阻塞 UI 线程
4.3 元数据处理
JPG 评级：通过 WIC 或 ExifTool 写入 Exif Rating 标签（0-5 星）
RAW 评级：通过 ExifTool 写入同名 .xmp 文件，遵循行业标准（如 Adobe XMP 规范）
批量处理：使用 ExifTool 批量命令行调用，提升元数据写入效率
4.4 文件系统操作
双路径加载：
扫描两个路径，按文件名匹配 JPG/RAW，生成 “虚拟图片项”（包含双格式文件信息）
路径失效时，提示用户重新选择或自动跳过无效路径
预删除管理：
维护一个预删除列表，记录所有标记为删除的虚拟图片项
缩略图渲染时，对预删除项添加红色角标
批量删除：
调用 Windows API 将文件移动到回收站，支持仅删除 JPG/RAW/ 全部
删除后更新缩略图区和预删除计数
导出功能：
按评级筛选虚拟图片项，复制对应文件到目标路径
自动创建 JPG 和 RAW 子文件夹，保持文件结构清晰
4.5 UI 交互设计
Fluent UI 组件：
标题栏：使用 CommandBar，按钮采用 Fluent 样式
预览区：使用 Image 控件，支持缩放和平移，多选时用 ItemsRepeater 平铺展示
缩略图区：使用 ItemsRepeater + 自定义 DataTemplate，显示缩略图、评级星标和文件类型标签
底部工具栏：使用 StackPanel，文件信息抽屉用 Expander 组件
交互反馈：
加载状态：显示进度条或加载动画
操作反馈：评级、删除、导出后即时更新 UI 状态
错误提示：路径无效、文件损坏时用 ContentDialog 弹窗提示
五、风险与应对
表格
风险	影响	应对措施
RAW 格式兼容性问题	部分相机 RAW 无法解码	优先依赖 WIC 原生支持，对罕见格式集成 ExifTool 作为 fallback
大数量图片加载卡顿	用户体验下降	严格实现懒加载和缓存，使用多线程异步处理，避免 UI 阻塞
元数据写入失败	评级丢失	写入前备份元数据，失败时提示用户并提供重试选项
路径失效导致崩溃	应用稳定性下降	增加路径有效性检查，失效时提示重新选择，不强制加载
