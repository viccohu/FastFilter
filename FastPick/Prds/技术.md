Windows 看图选片应用技术文档
一、项目概述
1.1 项目背景
本项目是为摄影工作流设计的 Windows 桌面应用，核心目标是高效筛选、评级和管理相机拍摄的 JPG+RAW 双格式照片，解决传统看图软件在双格式处理、批量评级和筛选上的痛点。
1.2 核心价值
双路径加载：同时加载两个文件夹，自动识别并折叠同一张照片的 JPG/RAW
高效评级：支持批量评级，JPG 写入 Exif，RAW 写入 .xmp 元数据
智能删除：预删除标记 + 批量删除，可选择仅删 JPG/RAW 或全部
快速导出：按评级筛选导出，自动分文件夹存放 JPG/RAW
流畅体验：基于 WIC 解码，支持 2000+ 张图片同时加载不卡顿
1.3 技术栈选型
UI 框架：WinUI 3 + Fluent UI，符合 Windows 设计语言，提供现代化界面
图片解码：Windows Imaging Component (WIC)，原生高效解码，支持常见 RAW 格式
元数据处理：ExifTool（成熟方案）或 WIC 原生 API，处理 JPG Exif 评级和 RAW .xmp 写入
文件系统：.NET System.IO + Windows API，处理双路径加载、批量操作和回收站删除
开发工具：Trae AI 代码助手 + Visual Studio，提升开发效率
二、架构设计
2.1 整体架构
采用 MVVM (Model-View-ViewModel) 架构，分离 UI 与业务逻辑，提升可维护性和测试性：
Model：文件系统操作、图片解码、元数据读写、批量处理核心逻辑
ViewModel：状态管理（选中图片、评级、预删除列表）、命令绑定（评级、删除、导出）
View：Fluent UI 组件，响应 ViewModel 状态变化，处理用户交互
2.2 核心模块划分

三、核心技术实现
3.1 图片加载与解码（WIC 服务）
3.1.1 核心类：WicImageService
职责：异步加载图片缩略图和预览图，扫描文件夹并匹配 JPG/RAW 双格式
关键实现：
支持常见图片格式：JPG、PNG、BMP 及 RAW 格式（CR2、NEF、ARW 等）
异步解码：使用 BitmapDecoder 异步加载，避免 UI 阻塞
双格式匹配：按文件名匹配 JPG 和 RAW，生成 “虚拟图片项”
性能优化：
懒加载：仅加载可见区域的缩略图，滚动时动态加载
缓存机制：缓存已解码的图片数据，避免重复解码

3.2 数据模型与状态管理
3.2.1 核心实体：PhotoItem
属性：文件名、文件路径、文件类型（JPG/RAW/ 双格式）、评级（0-5）、预删除标记、选中状态、缩略图 / 预览图源
特性：实现 INotifyPropertyChanged，支持 UI 实时更新
3.2.2 视图模型：MainViewModel
核心职责：
管理所有图片列表和筛选后列表
处理选中图片、评级、预删除等状态
调度图片加载、批量操作等业务逻辑
关键方法：
LoadPhotosAsync(string path1, string path2)：加载双路径图片
SetRatingForSelected(int rating)：批量设置选中图片评级
ToggleDeleteForSelected()：批量标记 / 取消预删除
UpdateDeleteCount()：更新预删除计数
3.3 UI 交互与布局
3.3.1 五段式布局
标题栏：使用 CommandBar 实现路径、导出、删除、筛选、设置按钮，支持抽屉式展开子选项
抽屉式展开区：与标题栏按钮联动，显示路径选择、导出配置、删除选项、筛选条件、设置项
预览区：支持单图预览（缩放、平移）和多选平铺（最多 5 张），使用 Image 和 ItemsRepeater 实现
缩略图区：使用 ItemsRepeater + UniformGridLayout 实现网格布局，支持横向滚动，显示缩略图、星级、文件类型、预删除角标
底部工具栏：包含文件信息抽屉、缩放控制、窗口控制，支持展开查看 Exif 信息和一键打开文件路径
3.3.2 交互设计
快捷键支持：0-5 数字键快速评级、Delete 键标记预删除、Ctrl+A 全选、Shift + 点击连续选择、Esc 关闭抽屉 / 弹窗
视觉反馈：选中状态蓝色高亮、预删除红色角标、操作成功 Toast 提示、加载状态进度环
四、核心功能实现
4.1 双路径加载
用户通过 “路径” 抽屉选择两个文件夹
WicImageService.ScanFolderAsync 扫描文件夹，匹配 JPG/RAW 生成 PhotoItem
MainViewModel.LoadPhotosAsync 合并双路径图片，更新 UI 列表
4.2 评级系统
用户选中图片，按 0-5 数字键或点击星级控件设置评级
MainViewModel.SetRatingForSelected 批量更新 PhotoItem.Rating
后续集成 ExifTool，将 JPG 评级写入 Exif，RAW 评级写入 .xmp 元数据
4.3 预删除功能
用户选中图片，按 Delete 键标记预删除
MainViewModel.ToggleDeleteForSelected 更新 PhotoItem.IsMarkedForDelete
删除按钮实时显示预删除数量，点击后弹出确认弹窗，支持仅删除 JPG/RAW/ 全部，默认放入回收站
4.4 筛选功能
用户通过 “筛选” 抽屉设置星级和文件类型条件
MainViewModel 根据条件过滤 FilteredPhotos 列表
缩略图区和预览区自动更新，筛选按钮高亮当前条件
4.5 导出功能
用户通过 “导出” 抽屉设置导出评级阈值、文件类型、导出路径
按评级筛选图片，复制对应文件到目标路径
自动创建 JPG/RAW 子文件夹，支持自定义导出文件夹名称
五、性能优化
5.1 图片加载优化
异步解码：使用 async/await 异步加载图片，避免 UI 阻塞
懒加载：仅加载可见区域的缩略图，滚动时动态加载新区域
缓存机制：缓存已解码的图片数据，避免重复解码
多线程：使用 Task 并行处理图片加载，提升加载速度
5.2 UI 渲染优化
虚拟化：使用 ItemsRepeater 虚拟化渲染缩略图，支持 2000+ 图片流畅滚动
数据绑定：使用 x:Bind 绑定数据，提升绑定性能
状态管理：通过 ViewModel 集中管理状态，避免频繁 UI 更新
六、风险与应对
表格
风险	影响	应对措施
RAW 格式兼容性问题	部分相机 RAW 无法解码	优先依赖 WIC 原生支持，对罕见格式集成 ExifTool 作为 fallback
大数量图片加载卡顿	用户体验下降	严格实现懒加载和缓存，使用多线程异步处理，避免 UI 阻塞
元数据写入失败	评级丢失	写入前备份元数据，失败时提示用户并提供重试选项
路径失效导致崩溃	应用稳定性下降	增加路径有效性检查，失效时提示重新选择，不强制加载
