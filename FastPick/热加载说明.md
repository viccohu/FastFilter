# FastPick 热加载 (Hot Reload) 使用说明

## 已配置的热加载方式

### 1. 命令行方式

#### 方式一：使用 dotnet run（推荐）
```bash
cd d:\PF2\FastPick
dotnet run --launch-profile "FastPick (Hot Reload)"
```

#### 方式二：使用 dotnet watch
```bash
cd d:\PF2\FastPick
dotnet watch run --launch-profile "FastPick (Hot Reload)"
```

### 2. VS Code 方式

#### 启动配置
在 VS Code 中按 `F5` 或 `Ctrl+Shift+D` 选择以下配置：
- **FastPick (Hot Reload)** - 带热加载的调试模式
- **FastPick (Debug)** - 普通调试模式

#### 任务配置
按 `Ctrl+Shift+P` 输入 `Tasks: Run Task` 选择：
- **hot-reload** - 启动热加载模式
- **watch** - 文件监视模式
- **build** - 仅编译

### 3. Visual Studio 方式

如果使用 Visual Studio 2022：
1. 打开项目
2. 点击调试按钮旁的下拉箭头
3. 选择 **FastPick (Hot Reload)** 启动配置
4. 点击调试按钮或按 `F5`

## 热加载支持的范围

### ✅ 支持热加载的内容
- XAML 文件修改（界面布局、样式）
- C# 代码文件（部分修改）
- 资源文件更新

### ❌ 不支持热加载的内容
- 项目文件 (.csproj) 修改
- 新增/删除文件
- 命名空间修改
- 类结构重大变更
- 构造函数签名修改

## 使用技巧

### 1. XAML 热加载
修改 XAML 文件后保存，界面会自动更新：
```xml
<!-- 修改前 -->
<TextBlock Text="FastPick" />

<!-- 修改后（保存后自动生效） -->
<TextBlock Text="FastPick Pro" FontSize="20" />
```

### 2. 代码热加载
修改代码后保存，部分逻辑会自动更新：
```csharp
// 可以热加载的修改
private void Button_Click(object sender, RoutedEventArgs e)
{
    // 修改这里的逻辑
    ShowToast("新的提示信息");
}
```

### 3. 强制刷新
如果热加载没有生效：
- 按 `Ctrl+Shift+F5` 重新加载
- 或在代码中添加 `// hotreload` 注释触发刷新

## 环境变量说明

| 变量 | 作用 |
|------|------|
| `DOTNET_HOTRELOAD=true` | 启用热加载 |
| `DOTNET_USE_POLLING_FILE_WATCHER=true` | 使用轮询文件监视器（更稳定） |

## 注意事项

1. **首次启动**仍需要完整编译
2. **XAML 修改**通常能立即生效
3. **C# 代码修改**有限制，复杂修改需要重启
4. 如果热加载失效，使用 `Ctrl+C` 停止后重新运行

## 快速启动脚本

创建 `start-hotreload.bat`：
```batch
@echo off
cd /d d:\PF2\FastPick
dotnet run --launch-profile "FastPick (Hot Reload)"
pause
```

双击即可启动带热加载的开发模式。
